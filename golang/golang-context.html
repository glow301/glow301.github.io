    Go 标准库学习 - context

    2020/10/11 Golang源码阅读

    # 什么是 Context

    根据官方的解释，context 是一个用于在 API 以及处理过程之间携带截止日期，取消信号以及其他请求的类型。我们可以在这里 (opens new window)看到关于 context 的完整解释。

    通俗地讲，context 有点类似 HTTP 中的 header。它可以设置一个超时时间，在函数的调用链中传递一些参数（一般只用于传递一些认证信息）。一般来说，我们在处理 HTTP 请求或者 RPC 的时候，经常会使用到 context。

    # Context 可以做什么

    context 最常用的用法，就是设置超时时间。在 go 语言中，每一个请求都会在单独的一个 goroutine 中处理，有时候我们还会开启额外的 goroutine 去请求 DB 或者进行其他 RPC 调用。我们通过在调用链或者 gortoutine 中传递 context，一旦 context 超时，或者被取消，其它所有的 gorountine 都可以收到信号，从而终止后续的流程。

    # 设置超时时间

    func main() {
	ctx, cancel := context.WithTimeout(context.Background(), 1*time.Second)
	defer cancel()

	go handle(ctx)

	select {
	case <-ctx.Done():
		fmt.Println("main: ", ctx.Err())
	}
}

func handle(ctx context.Context) {
	fmt.Println("in handle")
	time.Sleep(time.Second)
	fmt.Println("after sleep")
	time.Sleep(time.Second)
	fmt.Println("finish handle")
}

    在日常工作中，我们的函数调用可能会嵌套很多层，通过使用 context 可以方便的控制超时时间。

    # 传递认证信息

    context 的另一个常见用法是传递一些认证信息，或者类似 http header 传递一些元数据。

    func main() {
	ctx, cancel := context.WithTimeout(context.Background(), time.Second)
	defer cancel()

	ctx = context.WithValue(ctx, "userID", "123")

	go handle(ctx)
	select {
	case <-ctx.Done():
		fmt.Println("main: ", ctx.Err())
	}
}

func handle(ctx context.Context) {
	fmt.Println("value in ctx", ctx.Value("userID"))
}

    # 工作方式

    context 的所有代码都位于 src/context/context.go 文件中。除去注释可能只有 300-400 行。

    在我们研究 context 源码之前，我们先看看 context 包为我们暴露了哪些公有方法和属性。

    # 接口

    type Context interface {
    Deadline() (deadline time.Time, ok bool)
    Done() <-chan struct{}
    Err() error
    Value(key interface{}) interface{}
}

    context.Context 定义了 4 个方法。在上面的例子中，我们已经见过其中的一些方法。

    • Deadline 会返回 context 的截止日期（也就是 deadline），如果设置了 deadline（通过context.WithDeadline设置），ok 为 true，否则为 false。
    • Done 返回一个 channel，这个 channel 会在当前工作完成或 context 被取消后关闭。
      • 当 context 永远不会取消是，返回 nil。
      • 多次调用 Done 会返回同一个 channel。
    • Err 会返回一个 context 结束的原因。
      • 如果 Done 返回的 channel 没有被 close，也就是 context 没有结束，Err 返回 nil。
      • 如果 context 被 cancel，Err 返回 Canceled。
      • 如果 context 超时，Err 返回 DeadlineExceeded。
    • Value 从 context 中获取指定 key 的值，如果 key 没有被找到，返回 nil。

    # 公有函数

    WithCancel, WithDeadline, WithTimeout 和 WithValue 是 context 包为我们提供的 4 个包级别函数，它们的目的都是用于创建一个 context 或者为 context 设置 kv 键值对。

    # WithCancel

    WithCancel 会创建一个带有 cancel 函数的 context，我们可以通过返回的 cancel 函数来取消当前的 context。

    func main() {
	ctx, cancel := context.WithCancel(context.Background())

	go func() {
		time.Sleep(time.Second)
		cancel()
	}()
	go handle(ctx)

	select {
	case <-ctx.Done():
		fmt.Println("main: ", ctx.Err())

    # WithDeadline 和 WithTimeout

    WithDeadline 和 WithTimeout 这两个函数的功能其实是类似的，它们都是为 context 设置超时时间。唯一的区别在于，WithDeadline 是设置一个超时的时间点，而 WithTimeout 是设置过多久超时。

    func WithDeadline(parent Context, d time.Time) (Context, CancelFunc) {...}

func WithTimeout(parent Context, timeout time.Duration) (Context, CancelFunc) {
	return WithDeadline(parent, time.Now().Add(timeout))

    # 实现原理

    目前为止，我们对 context 包有了一个大致对了解，对于日常使用而言，应该是足够了。如果想要了解内部的实现原理，还需要进一步地分析代码。

    # WithCancel

    WithCancel 会返回一个 context 和一个实现了 cancel 方法的函数。

    func WithCancel(parent Context) (ctx Context, cancel CancelFunc) {
	c := newCancelCtx(parent)
	propagateCancel(parent, &c)
	return &c, func() { c.cancel(true, Canceled) }
}

    # newCancelCtx

    func newCancelCtx(parent Context) cancelCtx {
	return cancelCtx{Context: parent}
}

type cancelCtx struct {
	Context

	mu       sync.Mutex            // protects following fields
	done     chan struct{}         // created lazily, closed by first cancel call
	children map[canceler]struct{} // set to nil by the first cancel call
	err      error                 // set to non-nil by the first cancel call

    # propagateCancel

    func propagateCancel(parent Context, child canceler) {
	done := parent.Done()
	if done == nil {
		return // parent is never canceled
	}

	select {
	case <-done:
		// parent is already canceled
		child.cancel(false, parent.Err())
		return
	default:
	}

	if p, ok := parentCancelCtx(parent); ok {
		p.mu.Lock()
		if p.err != nil {
			// parent has already been canceled
			child.cancel(false, p.err)
		} else {
			if p.children == nil {
				p.children = make(map[canceler]struct{})
			}
			p.children[child] = struct{}{}
		}
		p.mu.Unlock()
	} else {
		atomic.AddInt32(&goroutines, +1)
		go func() {
			select {
			case <-parent.Done():
				child.cancel(false, parent.Err())
			case <-child.Done():
			}
		}()
	}
}

    propagateCancel 函数的目的是当父 context cancel 时，通知子 context。

    我们可以看到，通知是通过调用 child.cancel 函数实现的。在上面代码的所有分支中，有 3 处调用了child.cancel。

    1. 第一处调用是当 parent.Done 中有数据时，这说明 parent context 已经被 cancel 了，所以通知 child context cancel。
    2. 第二处调用是在 parentCancelCtx 返回的 p.err != nil 时，说明 parent context 已经被取消。另外要注意一点的是，如果当前 context 没有子 context，会将 children 属性置为空 strut。
    3. 第三处调用是在 parentCancelCtx 返回的 ok == false 时，当 parent.Done 触发时。

    注意最后的 goroutine，这里为每一个 context 开启了一个 goroutine，当 context 被取消时，这里的 select 就会收到 Done 的通知，继而 cancel 所有的子 context。

    这里 parentCancelCtx 函数的作用是检验 parent context 是不是和 cancelCtx 相匹配。它用到了多次调用 Done 返回相同的 channel。这里不用纠结，我们重点看 child.cancel 是如何传递信号的。

    func (c *cancelCtx) cancel(removeFromParent bool, err error) {
	if err == nil {
		panic("context: internal error: missing cancel error")
	}
	c.mu.Lock()
	if c.err != nil {
		c.mu.Unlock()
		return // already canceled
	}
	c.err = err
	if c.done == nil {
		c.done = closedchan
	} else {
		close(c.done)
	}
	for child := range c.children {
		// NOTE: acquiring the child's lock while holding parent's lock.
		child.cancel(false, err)
	}
	c.children = nil
	c.mu.Unlock()

	if removeFromParent {
		removeChild(c.Context, c)
	}
}

    # WithDeadline 和 WithTimeout

    之前我们看过，WithDeadline 和 WithTimeout 基本相同，我们只需要分析一个即可。这里与上面的 WithCancel 的区别在于多了一个超时限制。

    func WithDeadline(parent Context, d time.Time) (Context, CancelFunc) {
	if cur, ok := parent.Deadline(); ok && cur.Before(d) {
		// The current deadline is already sooner than the new one.
		return WithCancel(parent)
	}
	c := &timerCtx{
		cancelCtx: newCancelCtx(parent),
		deadline:  d,
	}
	propagateCancel(parent, c)
	dur := time.Until(d)
	if dur <= 0 {
		c.cancel(true, DeadlineExceeded) // deadline has already passed
		return c, func() { c.cancel(false, Canceled) }
	}
	c.mu.Lock()
	defer c.mu.Unlock()
	if c.err == nil {
		c.timer = time.AfterFunc(dur, func() {
			c.cancel(true, DeadlineExceeded)
		})
	}
	return c, func() { c.cancel(true, Canceled) }
}

    # defer

    到这里，细心的同学可能会有疑问，context 被 cancel 或者 timeout，里面的 defer 会执行吗？

    func main() {
	ctx, _ := context.WithTimeout(context.Background(), time.Second)

	go handle(ctx)

	select {
	case <-ctx.Done():
		fmt.Println("main: ", ctx.Err())

func handle(ctx context.Context) {

    可以发现，defer 中的函数并没有被执行。原因是这样的，按照官方的解释 (opens new window)，defer 会在下面 3 种情况下执行。

    1. 函数 return 时。
    2. 到达函数结尾处。
    3. 发生 panic。

    而 context 被取消或者 timeout 都不在这 3 种情况中，所以如果 context 被取消掉，里面的 defer 是不会被执行的。

    # 小结

    • context 为我们提供了在多个 goroutine 中设置超时，传递参数的一种方法。其中超时是我们使用比较多的方法。
    • 使用WithCancel, WithDeadline和WithTimeout来创建带有 cancel func 或者 timeout 的 context，使用context.Background创建空的 context。
    • 使用WithValue为 context 设置键值对，通过Value方法获取对应 key 的 value。
    • Done 方法返回一个 channel，这个 channel 会在 context 被 cancel 或者 timeout 后 close，如果 context 不会被取消，返回 nil。
    • Err 方法会返回 context 取消的原因，通常是Canceled或DeadlineExceeded。
    • 带有 cancel func 的 context 通过外部调用 cancel func 来取消 context，带有 timeout 的 context 内部通过time.AfterFunc来控制 timeout。
